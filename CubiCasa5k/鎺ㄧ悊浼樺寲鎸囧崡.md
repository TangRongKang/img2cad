# 推理优化指南 - 无需重新训练的提升方法

## 概述

对于已训练好的模型，有多种方法可以在**不重新训练**的情况下优化推理结果。本文档详细介绍这些方法及其参数调整。

## 一、测试时增强 (Test Time Augmentation, TTA)

### 1.1 旋转融合（已实现）

**原理**: 对图像进行多个角度旋转预测，然后平均结果。

**当前实现**: 4个旋转角度 `[(0, 0), (1, -1), (2, 2), (-1, 1)]`

**参数调整空间**:
- 可以增加更多旋转角度
- 可以只使用部分旋转角度（加快速度）

```python
# 当前: 4个旋转
rotations = [(0, 0), (1, -1), (2, 2), (-1, 1)]

# 可以扩展到8个旋转（更耗时但可能更准确）
rotations = [(0, 0), (1, -1), (2, 2), (-1, 1), 
             (0, 0), (1, -1), (2, 2), (-1, 1)]  # 重复
```

### 1.2 水平翻转增强

**原理**: 对图像进行水平翻转，预测后再翻转回来，与原始预测平均。

**优点**: 可以捕捉左右对称的模式，减少方向偏差。

**使用方法**:
```bash
python generate_color_map_advanced.py \
    --image image.png \
    --use-tta \
    --use-flip
```

### 1.3 多尺度预测

**原理**: 在不同尺度下预测，然后平均结果。

**适用场景**: 
- 小物体检测不佳时，使用较小尺度（0.8）
- 细节丢失时，使用较大尺度（1.2）

**使用方法**:
```bash
python generate_color_map_advanced.py \
    --image image.png \
    --use-tta \
    --use-scale \
    --scales 0.8 1.0 1.2
```

**参数说明**:
- `scales`: 尺度比例列表，例如 `[0.8, 1.0, 1.2]`
  - `<1.0`: 缩小，可以捕捉更大范围的上下文
  - `=1.0`: 原始尺寸（通常已包含）
  - `>1.0`: 放大，可以捕捉更多细节

### 1.4 TTA组合策略

**完全TTA** (最准确但最慢):
```bash
python generate_color_map_advanced.py \
    --image image.png \
    --use-tta \
    --use-flip \
    --use-scale \
    --scales 0.8 1.0 1.2
```
- 总预测次数: 4(旋转) × 2(翻转) + 2(额外尺度) = 10次
- 时间开销: 约10倍单次预测时间

**快速TTA** (平衡速度和准确性):
```bash
python generate_color_map_advanced.py \
    --image image.png \
    --use-tta \
    --use-flip
```
- 总预测次数: 4(旋转) × 2(翻转) = 8次
- 时间开销: 约8倍单次预测时间

## 二、后处理参数调整

### 2.1 多边形提取阈值

**参数**: `--polygon-threshold` (默认: 0.2)

**范围**: 0.1 - 0.5

**影响**:
- **较小值 (0.1-0.15)**: 
  - 检测更多细节和小的多边形
  - 可能产生更多噪声和误检
  - 适合：细节丰富的复杂户型图
  
- **中等值 (0.2-0.3)**: 
  - 平衡准确性和噪声
  - **推荐默认值**: 0.2
  
- **较大值 (0.4-0.5)**: 
  - 只保留高置信度的多边形
  - 可能丢失细节
  - 适合：噪声较多的图片

**使用方法**:
```bash
# 检测更多细节
python generate_color_map_advanced.py \
    --image image.png \
    --polygon-threshold 0.15

# 减少噪声
python generate_color_map_advanced.py \
    --image image.png \
    --polygon-threshold 0.3
```

### 2.2 概率阈值过滤

**参数**: `--prob-threshold` (默认: 0.0，即不启用)

**范围**: 0.0 - 1.0

**原理**: 如果预测的最大概率低于阈值，则将该像素设为Background类别。

**适用场景**:
- 模型输出过于"自信"，但实际有很多错误
- 想要更保守的预测，减少误检

**使用方法**:
```bash
# 只保留高置信度预测 (>0.5)
python generate_color_map_advanced.py \
    --image image.png \
    --prob-threshold 0.5

# 更保守的预测 (>0.7)
python generate_color_map_advanced.py \
    --image image.png \
    --prob-threshold 0.7
```

### 2.3 形态学平滑

**参数**: `--use-morphology` + `--morph-kernel` (默认: 3)

**原理**: 使用形态学操作（开运算+闭运算）去除小噪点和填充小洞。

**适用场景**:
- 预测结果有很多小噪点
- 房间边界不连续
- 想要更平滑的边界

**使用方法**:
```bash
# 轻度平滑 (kernel=3)
python generate_color_map_advanced.py \
    --image image.png \
    --use-morphology \
    --morph-kernel 3

# 中度平滑 (kernel=5)
python generate_color_map_advanced.py \
    --image image.png \
    --use-morphology \
    --morph-kernel 5

# 重度平滑 (kernel=7) - 可能丢失细节
python generate_color_map_advanced.py \
    --image image.png \
    --use-morphology \
    --morph-kernel 7
```

**注意**: 过大的kernel会丢失细节，特别是小的房间。

### 2.4 概率图平滑

**参数**: `--use-confidence-smooth` + `--smooth-kernel` (默认: 5)

**原理**: 对概率图进行平滑处理，再取argmax，可以得到更稳定的预测。

**优点**: 
- 减少边界抖动
- 使预测更加稳定

**缺点**:
- 可能使边界变模糊
- 增加计算时间

**使用方法**:
```bash
python generate_color_map_advanced.py \
    --image image.png \
    --use-confidence-smooth \
    --smooth-kernel 5
```

## 四、参数调优流程

### 步骤1: 基准测试

使用默认参数运行，观察结果:
```bash
python generate_color_map_advanced.py --image test.png
```

### 步骤2: 识别问题

观察输出图片，识别主要问题:
- **边界不清晰** → 使用概率平滑或形态学平滑
- **小物体丢失** → 降低多边形阈值，启用多尺度
- **噪声太多** → 提高多边形阈值，启用概率阈值
- **房间不连续** → 使用形态学平滑

### 步骤3: 逐步调整

每次只调整一个参数，观察效果:

```bash
# 先尝试调整多边形阈值
python generate_color_map_advanced.py --image test.png --polygon-threshold 0.25

# 如果还有问题，添加形态学平滑
python generate_color_map_advanced.py --image test.png --polygon-threshold 0.25 --use-morphology

# 继续添加其他优化
...
```

### 步骤4: 组合优化

找到最佳参数组合后，可以固定使用。

## 六、何时需要重新训练

以下情况建议重新训练，而不是仅调整推理参数:

1. **类别识别错误**: 模型总是将厨房识别为客厅
2. **结构性错误**: 模型完全无法识别某种房间类型
3. **新数据分布**: 图片风格与训练数据差异很大
4. **需要新类别**: 需要识别训练时没有的类别

**推理优化无法解决的情况**:
- 模型在训练数据上就没学好
- 数据分布差异太大
- 需要添加新的类别
