# hg_furukawa_original 模型结构说明

## 模型概述

`hg_furukawa_original` 是一个基于 **Hourglass（沙漏）架构**的深度卷积神经网络，用于户型图的分割任务。该模型能够同时预测：
- **21个热图通道**（heatmaps）：用于检测墙体端点、门窗位置等关键点
- **房间分割**：多个房间类别的语义分割
- **图标分割**：门窗、家具等图标的分割

---

## 整体架构

### 架构类型：编码器-解码器（Encoder-Decoder）

```
输入图像 (3通道, 512x512)
    ↓
编码器（下采样，提取特征）
    ↓
解码器（上采样，恢复分辨率）
    ↓
输出 (n_classes通道, 512x512)
```

---

## 核心组件

### 1. Residual（残差块）

**位置**: `Residual` 类（第7-51行）

**作用**: 基础特征提取单元，使用残差连接避免梯度消失

**结构**:
```
输入 x
  ↓
BatchNorm + ReLU
  ↓
Conv1x1 (降维到 numOut/2)
  ↓
BatchNorm + ReLU
  ↓
Conv3x3 (特征提取)
  ↓
BatchNorm + ReLU
  ↓
Conv1x1 (升维到 numOut)
  ↓
+ 残差连接 (如果输入输出维度不同，用Conv1x1调整)
  ↓
输出
```

**特点**:
- 如果 `numIn != numOut`，使用 `conv4` 调整残差连接的维度
- 使用 Kaiming 初始化

## 模型详细结构

### 阶段0：初始特征提取

```python
输入: (B, 3, H, W)  # B=batch, 3=RGB, H=高度, W=宽度

Conv2d(3→64, kernel=7, stride=2, padding=3)  # 下采样2倍
BatchNorm + ReLU
MaxPool2d(kernel=2, stride=2)  # 再下采样2倍，总共4倍

Residual(64→128)  # r01
Residual(128→128) # r02
Residual(128→128) # r03
Residual(128→256) # r04

输出: (B, 256, H/4, W/4)
```

**作用**: 将输入图像从原始尺寸下采样到1/4，提取基础特征

---

### 阶段1：第一个Hourglass单元

#### 编码路径（下采样）:
```python
输入: (B, 256, H/4, W/4)

MaxPool2d(kernel=2, stride=2)  # 下采样到 H/8, W/8
Residual(256→256) # r11_a
Residual(256→256) # r12_a
Residual(256→256) # r13_a

输出1a: (B, 256, H/8, W/8)
```

#### 解码路径（上采样）:
```python
输入: (B, 256, H/4, W/4)

Residual(256→256) # r11_b
Residual(256→256) # r12_b
Residual(256→512) # r13_b

输出1b: (B, 512, H/4, W/4)
```

**融合**: 
- 将 `输出1a` 上采样后与 `输出1b` 相加
- 使用 `_upsample_add` 方法处理尺寸不匹配

---

### 阶段2：第二个Hourglass单元

#### 编码路径:
```python
输入: 输出1a (B, 256, H/8, W/8)

MaxPool2d → (B, 256, H/16, W/16)
Residual(256→256) # r21_a
Residual(256→256) # r22_a
Residual(256→256) # r23_a

输出2a: (B, 256, H/16, W/16)
```

#### 解码路径:
```python
输入: 输出1a (B, 256, H/8, W/8)

Residual(256→256) # r21_b
Residual(256→256) # r22_b
Residual(256→512) # r23_b

输出2b: (B, 512, H/8, W/8)
```

**融合**: `输出2a` 上采样 + `输出2b` → 经过 `r2_` → 输出到阶段1

---

### 阶段3：第三个Hourglass单元

#### 编码路径:
```python
输入: 输出2a (B, 256, H/16, W/16)

MaxPool2d → (B, 256, H/32, W/32)
Residual(256→256) # r31_a
Residual(256→256) # r32_a
Residual(256→256) # r33_a

输出3a: (B, 256, H/32, W/32)
```

#### 解码路径:
```python
输入: 输出2a (B, 256, H/16, W/16)

Residual(256→256) # r31_b
Residual(256→256) # r32_b
Residual(256→512) # r33_b

输出3b: (B, 512, H/16, W/16)
```

**融合**: `输出3a` 上采样 + `输出3b` → 经过 `r3_` → 输出到阶段2

---

### 阶段4：最深层（瓶颈层）

#### 编码路径（最深层）:
```python
输入: 输出3a (B, 256, H/32, W/32)

MaxPool2d → (B, 256, H/64, W/64)
Residual(256→256) # r41_a
Residual(256→256) # r42_a
Residual(256→256) # r43_a
Residual(256→512) # r44_a
Residual(512→512) # r45_a

输出4a: (B, 512, H/64, W/64)  # 最深层特征
```

#### 解码路径:
```python
输入: 输出3a (B, 256, H/32, W/32)

Residual(256→256) # r41_b
Residual(256→256) # r42_b
Residual(256→512) # r43_b

输出4b: (B, 512, H/32, W/32)
```

**融合**: `输出4a` 上采样 + `输出4b` → 经过 `r4_` → 输出到阶段3

---

### 最终输出层

```python
输入: 融合后的特征 (B, 512, H/4, W/4)

Conv2d(512→512, kernel=1) + BatchNorm + ReLU
Conv2d(512→256, kernel=1) + BatchNorm + ReLU
Conv2d(256→n_classes, kernel=1)  # n_classes通常是44

ConvTranspose2d(n_classes→n_classes, kernel=4, stride=4)  # 上采样4倍回到原始尺寸

# 前21个通道通过Sigmoid（热图）
out[:, :21] = sigmoid(out[:, :21])

输出: (B, n_classes, H, W)
```

## 关键特性

### 1. Hourglass架构

模型使用**多个嵌套的Hourglass单元**，每个单元包含：
- **编码路径**：下采样提取深层特征
- **解码路径**：上采样恢复分辨率
- **跳跃连接**：将编码路径的特征与解码路径融合

**优势**:
- 能够捕获多尺度特征
- 通过跳跃连接保留细节信息
- 适合密集预测任务（如分割）

### 2. 多尺度特征融合

```
阶段1 (H/8)  ←→  阶段2 (H/16)  ←→  阶段3 (H/32)  ←→  阶段4 (H/64)
    ↑              ↑              ↑              ↑
    └──────────────┴──────────────┴──────────────┘
             通过上采样和相加融合
```

每个阶段都融合了不同尺度的特征，使模型能够：
- 检测大物体（房间）
- 检测小物体（门窗、家具）
- 保持边界精度

### 3. 输出处理

**前21个通道（热图）**:
- 使用 `Sigmoid` 激活
- 输出范围 [0, 1]
- 用于检测关键点（墙体端点、门窗位置等）

**后23个通道（分割）**:
- 不使用激活函数（在推理时使用Softmax）
- 用于房间和图标的分割

## 关键方法

### `_upsample_add` (第210-230行)

**作用**: 上采样并相加两个特征图

**为什么需要**:
- PyTorch的上采样可能因为奇数尺寸导致尺寸不匹配
- 使用 `bilinear` 插值确保尺寸匹配

**实现**:
```python
def _upsample_add(self, x, y):
    _, _, H, W = y.size()
    if y.shape != x.shape:
        # 使用双线性插值上采样x到y的尺寸
        return F.interpolate(x, size=(H, W), mode='bilinear', align_corners=False) + y
    else:
        return x + y
```

## 从模型输出到色块特征图

### 完整流程

模型输出后，需要经过以下步骤才能得到可视化的色块特征图：

```
模型输出 (B, 44, H, W)
    ↓
1. 分割输出通道
    ↓
2. 提取房间类别部分
    ↓
3. Softmax得到概率
    ↓
4. Argmax得到类别ID
    ↓
5. 类别ID映射到颜色
    ↓
色块特征图 (H, W, 3)
```

---

### 详细步骤

#### 步骤1: 模型输出

模型输出形状：`(B, n_classes, H, W)`
- `B`: batch size（通常为1）
- `n_classes`: 44（总类别数）
- `H, W`: 图像高度和宽度（通常512x512）

**输出通道分配**（`split=[21, 12, 11]`）：
- **前21个通道**：热图（heatmaps）
  - 用于检测墙体端点、门窗位置等关键点
  - 已经通过Sigmoid激活，值域[0, 1]
- **中间12个通道**：房间类别（rooms）
  - 12个房间类别：Background, Outdoor, Wall, Kitchen, Living Room, Bed Room, Bath, Entry, Railing, Storage, Garage, Undefined
  - 原始输出是logits（未激活）
- **后11个通道**：图标类别（icons）
  - 用于检测门窗、家具等图标

#### 步骤2: 提取房间类别部分

```python
# 从完整输出中提取房间部分
rooms_logits = prediction[:, 21:21+12, :, :]  # (1, 12, H, W)
rooms_logits = rooms_logits.squeeze(0)  # (12, H, W)
```

**说明**：
- `prediction[:, 21:33, :, :]` 提取中间12个通道
- 这12个通道对应12个房间类别

#### 步骤3: 应用优化（可选）

在得到最终类别之前，可以应用各种优化：

**3.1 类别权重调整**：
```python
# 降低Undefined类别的权重
class_weights = {11: 0.3}  # Undefined类别权重设为0.3
rooms_logits = apply_class_weights(rooms_logits, class_weights)
```

**3.2 温度缩放**：
```python
# 调整softmax的"温度"
rooms_logits = apply_temperature_scaling(rooms_logits, temperature=0.9)
```

#### 步骤4: Softmax得到概率分布

```python
# 对每个像素位置，计算12个类别的概率分布
rooms_prob = F.softmax(rooms_logits, dim=0)  # (12, H, W)
```

**结果**：
- 每个像素位置有12个概率值
- 所有概率值之和为1
- 形状：`(12, H, W)`

#### 步骤5: Argmax得到类别ID

```python
# 对每个像素，选择概率最大的类别
rooms_pred = np.argmax(rooms_prob, axis=0)  # (H, W)
```

**结果**：
- 每个像素对应一个类别ID（0-11）
- 形状：`(H, W)`，值为整数

**示例**：
```python
# 某个像素的概率分布
prob = [0.1, 0.05, 0.02, 0.3, 0.4, 0.05, 0.03, 0.02, 0.01, 0.01, 0.01, 0.0]
# argmax后得到类别ID = 4 (Living Room)
```

#### 步骤6: 类别ID映射到颜色

```python
# 定义类别到颜色的映射
room_colors = {
    0: (220, 220, 220),      # Background - 浅灰色
    1: (179, 222, 105),      # Outdoor - 浅绿色
    2: (0, 0, 0),            # Wall - 黑色
    3: (141, 211, 199),      # Kitchen - 青色
    4: (253, 180, 98),       # Living Room - 橙色
    5: (252, 205, 229),      # Bed Room - 粉色
    6: (128, 177, 211),      # Bath - 蓝色
    7: (128, 128, 128),      # Entry - 灰色
    8: (255, 255, 179),      # Railing - 黄色
    9: (190, 186, 218),      # Storage - 紫色
    10: (251, 128, 114),     # Garage - 红色
    11: (255, 255, 255),     # Undefined - 白色
}

# 创建RGB图像
color_map = np.zeros((H, W, 3), dtype=np.uint8)

# 为每个类别ID分配颜色
for class_id in range(12):
    mask = (rooms_pred == class_id)
    color_map[mask] = room_colors[class_id]
```

**结果**：
- 形状：`(H, W, 3)`，RGB图像
- 每个像素的颜色对应其类别

#### 步骤7: 调整尺寸（如果需要）

如果输入图像被缩放，需要恢复到原始尺寸：

```python
if original_size is not None:
    color_map = cv2.resize(
        color_map, 
        (original_size[1], original_size[0]),  # (width, height)
        interpolation=cv2.INTER_NEAREST  # 使用最近邻保持类别边界清晰

```

**注意**：必须使用`INTER_NEAREST`（最近邻插值），不能使用双线性插值，否则会混合不同类别的颜色。

### 完整代码示例

```python
import torch
import torch.nn.functional as F
import numpy as np
import cv2

# 1. 模型输出
prediction = model(img_tensor)  # (1, 44, 512, 512)

# 2. 提取房间类别部分
split = [21, 12, 11]  # [热图, 房间, 图标]
rooms_logits = prediction[:, split[0]:split[0]+split[1], :, :]  # (1, 12, 512, 512)
rooms_logits = rooms_logits.squeeze(0)  # (12, 512, 512)

# 3. Softmax得到概率
rooms_prob = F.softmax(rooms_logits, dim=0)  # (12, 512, 512)
rooms_prob_np = rooms_prob.cpu().numpy()

# 4. Argmax得到类别ID
rooms_pred = np.argmax(rooms_prob_np, axis=0)  # (512, 512)

# 5. 映射到颜色
room_colors = {
    0: (220, 220, 220),   # Background
    1: (179, 222, 105),   # Outdoor
    2: (0, 0, 0),         # Wall
    3: (141, 211, 199),   # Kitchen
    4: (253, 180, 98),    # Living Room
    5: (252, 205, 229),   # Bed Room
    6: (128, 177, 211),   # Bath
    7: (128, 128, 128),   # Entry
    8: (255, 255, 179),   # Railing
    9: (190, 186, 218),   # Storage
    10: (251, 128, 114),  # Garage
    11: (255, 255, 255),  # Undefined
}

H, W = rooms_pred.shape
color_map = np.zeros((H, W, 3), dtype=np.uint8)

for class_id in range(12):
    mask = (rooms_pred == class_id)
    color_map[mask] = room_colors[class_id]

# 6. 保存图像
cv2.imwrite('color_map.png', cv2.cvtColor(color_map, cv2.COLOR_RGB2BGR))
```

### 与多边形分割的区别

**色块特征图**：
- 直接使用argmax结果
- 每个像素对应一个类别
- 边界可能不够平滑

**多边形分割**：
- 使用后处理算法提取多边形
- 边界更平滑、更符合实际房间形状
- 需要额外的后处理步骤（见`create_polygon_segmentation`函数）

两种方法可以结合使用：
1. 色块图用于快速预览和调试
2. 多边形图用于最终输出和进一步处理

## 矢量多边形分割详解

### 概述

矢量多边形分割是一种后处理技术，将像素级的分割结果转换为矢量多边形，具有以下优势：
- **边界平滑**：多边形边界比像素边界更平滑
- **符合实际**：房间通常是矩形或多边形，而不是像素块
- **易于处理**：矢量格式便于后续处理（如CAD导入、面积计算等）
- **文件更小**：矢量格式比像素图占用更少空间

---

### 完整流程

```
模型输出 (1, 44, H, W)
    ↓
1. 分割输出通道
    ↓
2. 提取墙体、房间、图标
    ↓
3. 从热图提取墙体端点
    ↓
4. 构建墙体线段
    ↓
5. 计算交点（junction points）
    ↓
6. 生成网格多边形（grid polygons）
    ↓
7. 为每个多边形分配房间类别
    ↓
8. 合并相邻矩形
    ↓
9. 提取门窗多边形
    ↓
10. 渲染为图像
    ↓
矢量多边形分割图 (H, W)
```

### 详细步骤

#### 步骤1: 分割输出通道


4. 生成所有可能的矩形组合

**示例**：

```python

grid_polygons_new = []

        grid_polygons_new.append(polygon)
        room_types.append({'type': 'room', 'class': room_class})
```

**`get_polygon_class` 函数**：
```python
def get_polygon_class(polygon, segmentation):
    # 在多边形区域内绘制掩码
    rr, cc = draw.polygon(polygon[:, 1], polygon[:, 0])  # 注意y, x顺序
    
    # 获取多边形内的所有像素值
    area = segmentation[rr, cc]
    
    # 统计类别分布
    values, counts = np.unique(area, return_counts=True)
    
    # 选择出现次数最多的类别
    if len(counts) > 0:
        ind = np.argmax(counts)
        return values[ind]
    return None
```

**类别分配逻辑**：
- 在多边形区域内统计所有像素的类别
- 选择出现次数最多的类别作为该多边形的类别
- 如果区域内主要是Background，可能返回None（被过滤）

#### 步骤6: 合并相邻矩形


**合并逻辑**：
1. 使用Shapely库检测相邻矩形
2. 如果两个矩形类别相同且相邻，合并为一个多边形
3. 使用`unary_union`合并几何形状

**优势**：
- 减少多边形数量
- 生成更符合实际房间形状的多边形
- 避免同一房间被分割成多个小矩形

**`polygons_to_image` 函数**：
```python
def polygons_to_image(polygons, types, room_polygons, room_types, height, width):
    # 创建两个分割图像
    room_seg = np.zeros((12, height, width), dtype=np.float32)
    icon_seg = np.zeros((11, height, width), dtype=np.float32)
    
    # 渲染房间多边形
    for polygon, room_type in zip(room_polygons, room_types):
        class_id = room_type['class']
        rr, cc = draw.polygon(polygon[:, 1], polygon[:, 0])
        # 裁剪到图像范围内
        mask = (rr >= 0) & (rr < height) & (cc >= 0) & (cc < width)
        rr, cc = rr[mask], cc[mask]
        room_seg[class_id, rr, cc] = 1.0
    
    # 渲染其他多边形（墙体、图标、门窗）
    for polygon, poly_type in zip(polygons, types):
        if poly_type['type'] == 'icon':
            class_id = poly_type['class']
            rr, cc = draw.polygon(polygon[:, 1], polygon[:, 0])
            mask = (rr >= 0) & (rr < height) & (cc >= 0) & (cc < width)
            rr, cc = rr[mask], cc[mask]
            icon_seg[class_id, rr, cc] = 1.0
    
    # 转换为类别ID
    room_seg_2D = np.argmax(room_seg, axis=0)
    icon_seg_2D = np.argmax(icon_seg, axis=0)
    
    return room_seg_2D, icon_seg_2D
```

---

### 关键算法

#### 1. 局部最大值提取（extract_local_max）

**作用**：从热图中提取关键点

**算法**：
1. 对热图应用阈值过滤
2. 使用非极大值抑制（NMS）去除重复点
3. 返回局部最大值点的坐标和置信度

#### 2. 点连接（calc_point_info）

**作用**：将关键点连接成线段

**算法**：
1. 根据点的方向和距离判断是否应该连接
2. 考虑点的方向性（上下左右）
3. 使用gap参数控制连接距离

#### 3. 矩形合并（merge_rectangles）

**作用**：合并相邻的相同类别矩形

**算法**：
1. 使用Shapely检测几何相邻关系
2. 如果两个矩形类别相同且相邻，合并
3. 使用`unary_union`生成合并后的多边形

#### 4. 重叠移除（remove_overlapping_openings）

**作用**：移除重叠的门窗

**算法**：
1. 计算门窗之间的IoU（交并比）
2. 如果IoU > 阈值，保留面积更大的
3. 移除面积较小的重叠门窗

---

### 参数说明

#### threshold（阈值）

**作用**：控制关键点提取的敏感度

**取值范围**：0.0 - 1.0

**影响**：
- **较低值（0.1-0.2）**：提取更多关键点，可能包含噪声
- **较高值（0.3-0.5）**：只提取高置信度点，可能遗漏一些点

**推荐值**：0.2（平衡精度和召回率）

#### opening_types（开口类型）

**作用**：指定要提取的开口类型

**可选值**：
- `[1]`: 只提取窗户
- `[2]`: 只提取门
- `[1, 2]`: 提取窗户和门（推荐）

### 常见问题

**Q: 为什么多边形图和色块图不一致？**

A: 可能的原因：
1. 多边形提取时使用了不同的概率分布
2. 小区域在多边形提取时被过滤或合并
3. 类别分配时使用了不同的统计方法（像素计数 vs 面积占比）

**解决方案**：
- 确保使用相同的优化概率分布
- 调整`threshold`参数
- 检查`get_polygon_class`的类别分配逻辑

**Q: 如何提高多边形提取质量？**

A: 
1. 提高模型预测质量（使用TTA、优化参数等）
2. 调整`threshold`参数（通常0.15-0.25效果较好）
3. 检查热图质量（确保关键点检测准确）
4. 使用优化后的概率分布（而不是原始预测）

## 色块边界平滑方案

### 概述

直接对色块特征图（像素级分割结果）进行后处理，生成边界平滑的色块，**无需复杂的多边形提取流程**。

### 方案对比

| 方案 | 优点 | 缺点 | 速度 | 推荐度 |
|------|------|------|------|--------|
| **形态学操作** | 简单、快速 | 可能改变区域大小 | ⚡⚡⚡ 快 | ⭐⭐⭐⭐⭐ |
| **中值滤波** | 保持边缘 | 可能模糊细节 | ⚡⚡⚡ 快 | ⭐⭐⭐⭐ |
| **综合方案** | 效果好 | 需要调参 | ⚡⚡ 中等 | ⭐⭐⭐⭐⭐ |
| **CRF** | 效果最好 | 计算复杂、需要额外库 | ⚡ 慢 | ⭐⭐⭐ |

---

### 方案1: 形态学操作（推荐）

**原理**：使用开运算和闭运算平滑边界
- **开运算**：去除小噪点，平滑凸出边界
- **闭运算**：填充小孔洞，平滑凹陷边界

**实现**：
```python
def smooth_segmentation_morphology(segmentation, kernel_size=3, iterations=1):
    smoothed = segmentation.copy().astype(np.uint8)
    
    for class_id in np.unique(segmentation):
        if class_id == 0:  # 跳过Background
            continue
        
        mask = (segmentation == class_id).astype(np.uint8)
        kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (kernel_size, kernel_size))
        
        # 开运算：去除小噪点
        mask = cv2.morphologyEx(mask, cv2.MORPH_OPEN, kernel, iterations=iterations)
        # 闭运算：填充小孔洞
        mask = cv2.morphologyEx(mask, cv2.MORPH_CLOSE, kernel, iterations=iterations)
        
        smoothed[mask == 1] = class_id
    
    return smoothed
```

**参数**：
- `kernel_size=3`: 轻微平滑（推荐）
- `kernel_size=5`: 中等平滑
- `kernel_size=7`: 强平滑

---

### 方案2: 中值滤波

**原理**：使用中值滤波去除孤立噪点，保持边缘清晰

**实现**：
```python
from scipy.ndimage import median_filter

def smooth_segmentation_median(segmentation, kernel_size=5):
    smoothed = median_filter(segmentation.astype(np.float32), size=kernel_size)
    return smoothed.astype(np.int32)
```

**特点**：
- ✅ 保持边缘清晰
- ✅ 去除孤立噪点
- ⚠️ 核大小需要仔细选择

---

### 方案3: 综合方案（最佳平衡）

**原理**：结合形态学、中值滤波和孔洞填充

**实现**：
```python
def smooth_segmentation_advanced(segmentation, probability_map=None, 
                                 morph_kernel=3, morph_iterations=1,
                                 median_kernel=5, fill_holes=True):
    smoothed = segmentation.copy()
    
    # 1. 形态学平滑
    smoothed = smooth_segmentation_morphology(smoothed, morph_kernel, morph_iterations)
    
    # 2. 中值滤波去除孤立点
    smoothed = smooth_segmentation_median(smoothed, median_kernel)
    
    # 3. 填充小孔洞
    if fill_holes:
        from scipy.ndimage import binary_fill_holes
        for class_id in np.unique(smoothed):
            if class_id == 0:
                continue
            mask = (smoothed == class_id).astype(bool)
            filled_mask = binary_fill_holes(mask)
            smoothed[filled_mask & (smoothed == 0)] = class_id
    
    return smoothed
```

1. **保持类别一致性**：平滑后确保类别ID仍然有效（0-11）
2. **避免过度平滑**：可能丢失小区域或细节
3. **参数调整**：根据实际效果调整`kernel_size`
4. **性能考虑**：形态学操作最快，适合实时处理

### 与多边形分割的对比

| 特性 | 色块平滑 | 多边形分割 |
|------|----------|------------|
| **实现复杂度** | 简单 | 复杂 |
| **计算速度** | 快 | 慢 |
| **边界质量** | 平滑 | 很平滑 |
| **适用场景** | 快速预览、实时处理 | 最终输出、CAD导入 |
| **文件格式** | 像素图 | 矢量图 |

**建议**：
- **开发调试**：使用色块平滑（快速）
- **最终输出**：使用多边形分割（高质量）
- **实时处理**：使用色块平滑（快速）
