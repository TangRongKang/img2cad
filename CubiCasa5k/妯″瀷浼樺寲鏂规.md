# hg_furukawa_original 模型优化方案

## 当前模型架构分析

### 架构特点
- **Hourglass（沙漏）架构**：4个Hourglass单元，编码器-解码器结构
- **深度**：非常深（多个残差块堆叠）
- **多尺度特征融合**：通过 `_upsample_add` 融合不同尺度的特征
- **多任务输出**：同时预测热图（21通道）、房间分割（12通道）、图标分割（11通道）

### 潜在问题
1. **模型深度过深**：可能导致梯度消失/爆炸
2. **参数量大**：512通道的特征图占用大量内存
3. **计算量大**：多个Hourglass单元重复计算
4. **特征融合方式简单**：仅使用加法融合，可能丢失信息

---

## 优化方案

### 1. 架构优化

#### 1.1 深度可分离卷积（Depthwise Separable Convolution）

**问题**：标准卷积计算量大，参数量多

**优化**：使用深度可分离卷积减少参数量和计算量

```python
# 原始代码
self.conv2 = nn.Conv2d(int(self.numOut / 2), int(self.numOut / 2),
                       bias=True, kernel_size=3, stride=1, padding=1)

# 优化后
class DepthwiseSeparableConv(nn.Module):
    def __init__(self, in_channels, out_channels, kernel_size=3, stride=1, padding=1):
        super().__init__()
        self.depthwise = nn.Conv2d(in_channels, in_channels, kernel_size, 
                                   stride, padding, groups=in_channels)
        self.pointwise = nn.Conv2d(in_channels, out_channels, 1)
    
    def forward(self, x):
        x = self.depthwise(x)
        x = self.pointwise(x)
        return x

# 在Residual中使用
self.conv2 = DepthwiseSeparableConv(int(self.numOut / 2), int(self.numOut / 2))
```

**效果**：
- 参数量减少约 **8-9倍**
- 计算量减少约 **8-9倍**
- 精度损失通常 < 1%

#### 1.2 注意力机制（Attention Mechanism）

**问题**：特征融合方式简单，没有选择性关注重要特征

**优化**：添加通道注意力和空间注意力

```python
class ChannelAttention(nn.Module):
    def __init__(self, in_channels, reduction=16):
        super().__init__()
        self.avg_pool = nn.AdaptiveAvgPool2d(1)
        self.max_pool = nn.AdaptiveMaxPool2d(1)
        self.fc = nn.Sequential(
            nn.Conv2d(in_channels, in_channels // reduction, 1),
            nn.ReLU(),
            nn.Conv2d(in_channels // reduction, in_channels, 1),
            nn.Sigmoid()
        )
    
    def forward(self, x):
        avg_out = self.fc(self.avg_pool(x))
        max_out = self.fc(self.max_pool(x))
        return x * (avg_out + max_out)

class SpatialAttention(nn.Module):
    def __init__(self, kernel_size=7):
        super().__init__()
        self.conv = nn.Conv2d(2, 1, kernel_size, padding=kernel_size//2)
        self.sigmoid = nn.Sigmoid()
    
    def forward(self, x):
        avg_out = torch.mean(x, dim=1, keepdim=True)
        max_out, _ = torch.max(x, dim=1, keepdim=True)
        x = torch.cat([avg_out, max_out], dim=1)
        x = self.conv(x)
        return x * self.sigmoid(x)

class CBAM(nn.Module):
    """Convolutional Block Attention Module"""
    def __init__(self, in_channels, reduction=16):
        super().__init__()
        self.ca = ChannelAttention(in_channels, reduction)
        self.sa = SpatialAttention()
    
    def forward(self, x):
        x = self.ca(x)
        x = self.sa(x)
        return x

# 在特征融合时使用
def _upsample_add(self, x, y):
    x = F.interpolate(x, size=y.shape[2:], mode='bilinear', align_corners=False)
    # 添加注意力机制
    fused = x + y
    attention = self.cbam(fused)  # 在融合后应用注意力
    return attention
```

**效果**：
- 提高特征融合质量
- 关注重要特征，抑制噪声
- 精度提升约 **2-3%**

#### 1.3 特征金字塔网络（FPN）改进

**问题**：当前的特征融合方式可能丢失细节信息

**优化**：使用更复杂的特征融合方式

```python
class FPNFusion(nn.Module):
    def __init__(self, in_channels):
        super().__init__()
        self.conv = nn.Conv2d(in_channels * 2, in_channels, 1)
        self.bn = nn.BatchNorm2d(in_channels)
        self.relu = nn.ReLU()
    
    def forward(self, x, y):
        # 上采样x到y的尺寸
        x = F.interpolate(x, size=y.shape[2:], mode='bilinear', align_corners=False)
        # 拼接后融合
        fused = torch.cat([x, y], dim=1)
        fused = self.conv(fused)
        fused = self.bn(fused)
        fused = self.relu(fused)
        return fused

# 替换原来的 _upsample_add
def _upsample_add(self, x, y):
    return self.fpn_fusion(x, y)
```

**效果**：
- 更好地保留细节信息
- 精度提升约 **1-2%**

#### 1.4 减少Hourglass单元数量

**问题**：4个Hourglass单元计算量大，可能过度设计

**优化**：减少到2-3个Hourglass单元，或使用更轻量的设计

```python
# 方案1：减少Hourglass数量（从4个减少到2个）
# 方案2：使用更轻量的Hourglass设计
class LightweightHourglass(nn.Module):
    """轻量级Hourglass单元，减少残差块数量"""
    def __init__(self, in_channels):
        super().__init__()
        # 减少残差块数量：从3个减少到2个
        self.down1 = Residual(in_channels, in_channels)
        self.down2 = Residual(in_channels, in_channels)
        self.up1 = Residual(in_channels, in_channels)
        self.up2 = Residual(in_channels, in_channels)
    
    def forward(self, x):
        # 简化的Hourglass结构
        down = self.down2(self.down1(x))
        up = self.up2(self.up1(down))
        return up
```

**效果**：
- 计算量减少约 **30-40%**

### 2. 训练优化

#### 2.1 知识蒸馏（Knowledge Distillation）

**问题**：大模型训练时间长，推理慢

**优化**：使用大模型（教师）指导小模型（学生）训练

```python
class KnowledgeDistillationLoss(nn.Module):
    def __init__(self, temperature=4.0, alpha=0.7):
        super().__init__()
        self.temperature = temperature
        self.alpha = alpha
        self.kl_div = nn.KLDivLoss(reduction='batchmean')
        self.ce_loss = nn.CrossEntropyLoss()
    
    def forward(self, student_logits, teacher_logits, labels):
        # 软标签损失（教师模型的输出）
        soft_loss = self.kl_div(
            F.log_softmax(student_logits / self.temperature, dim=1),
            F.softmax(teacher_logits / self.temperature, dim=1)
        ) * (self.temperature ** 2)
        
        # 硬标签损失（真实标签）
        hard_loss = self.ce_loss(student_logits, labels)
        
        # 组合损失
        return self.alpha * soft_loss + (1 - self.alpha) * hard_loss
```

**效果**：
- 小模型可以达到大模型 **90-95%** 的精度
- 推理速度提升 **2-3倍**

#### 3.2 模型剪枝（Pruning）

**问题**：模型中有很多冗余参数

**优化**：剪枝不重要的通道或权重

```python
import torch.nn.utils.prune as prune

# 结构化剪枝（按通道）
def prune_model(model, pruning_ratio=0.3):
    for module in model.modules():
        if isinstance(module, nn.Conv2d):
            prune.ln_structured(module, name='weight', amount=pruning_ratio, 
                              n=2, dim=0)  # L2范数，按通道剪枝
            prune.remove(module, 'weight')
    return model

# 非结构化剪枝（按权重）
def unstructured_prune(model, pruning_ratio=0.5):
    for module in model.modules():
        if isinstance(module, (nn.Conv2d, nn.Linear)):
            prune.random_unstructured(module, name='weight', amount=pruning_ratio)
            prune.remove(module, 'weight')
    return model
```

**效果**：
- 模型大小减少约 **30-50%**
- 推理速度提升约 **1.5-2倍**
- 需要重新训练以恢复精度

### 4. 损失函数优化

#### 4.1 Focal Loss（用于类别不平衡）

**问题**：Background类别占大多数，导致类别不平衡

**优化**：使用Focal Loss关注难分类样本

```python
class FocalLoss(nn.Module):
    def __init__(self, alpha=1, gamma=2):
        super().__init__()
        self.alpha = alpha
        self.gamma = gamma
    
    def forward(self, inputs, targets):
        ce_loss = F.cross_entropy(inputs, targets, reduction='none')
        pt = torch.exp(-ce_loss)
        focal_loss = self.alpha * (1 - pt) ** self.gamma * ce_loss
        return focal_loss.mean()
```

**效果**：
- 提高小类别（如Entry、Storage）的检测精度
- 整体精度提升约 **1-2%**

---

#### 4.2 Dice Loss（用于分割任务）

**问题**：交叉熵损失可能不适合分割任务

**优化**：结合Dice Loss

```python
class DiceLoss(nn.Module):
    def __init__(self, smooth=1.0):
        super().__init__()
        self.smooth = smooth
    
    def forward(self, pred, target):
        pred = F.softmax(pred, dim=1)
        target_one_hot = F.one_hot(target, num_classes=pred.shape[1]).permute(0, 3, 1, 2)
        
        pred_flat = pred.view(-1)
        target_flat = target_one_hot.view(-1)
        
        intersection = (pred_flat * target_flat).sum()
        dice = (2. * intersection + self.smooth) / (pred_flat.sum() + target_flat.sum() + self.smooth)
        return 1 - dice

# 组合损失
combined_loss = 0.5 * ce_loss + 0.5 * dice_loss
```

**效果**：
- 提高分割边界精度
- IoU提升约 **2-3%**
