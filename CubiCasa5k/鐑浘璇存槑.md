# 热图（Heatmap）说明

## 什么是热图？

**热图（Heatmap）** 是模型输出的21个通道，用于检测户型图中的**关键点位置**。每个通道对应一种特定类型的关键点，值域在 [0, 1] 之间，表示该位置存在该类型关键点的**概率**。

## 热图的21个通道

### 通道 0-12：墙体相关关键点（13个通道）

这些通道用于检测墙体的关键点，包括：
- **墙体端点**：墙体的起点和终点
- **墙体角点**：墙体转折处的角点
- **墙体交点**：多条墙体相交的交点

**用途**：从这些关键点可以构建墙体线段，进而生成墙体多边形。

```python
# 在 post_prosessing.py 中
wall_heatmaps = heatmaps[:13]  # 使用前13个通道提取墙体
```

### 通道 13-16：门窗角点（4个通道）

用于检测门窗（opening）的四个角点：
- **通道 13**：门窗的**左角点**（left corner）
- **通道 14**：门窗的**右角点**（right corner）
- **通道 15**：门窗的**上角点**（up corner）
- **通道 16**：门窗的**下角点**（down corner）

**用途**：通过这些角点可以定位门窗的位置和大小。

### 通道 17-20：图标角点（4个通道）

用于检测图标（icon，如家具、设备）的四个角点：
- **通道 17**：图标的**左上角点**（upper_left corner）
- **通道 18**：图标的**右上角点**（upper_right corner）
- **通道 19**：图标的**左下角点**（lower_left corner）
- **通道 20**：图标的**右下角点**（lower_right corner）

**用途**：通过这些角点可以定位图标的位置和边界框。

## 热图的生成过程

### 1. 标注阶段（从SVG文件）

在数据标注时，从SVG文件中提取关键点坐标：

```python
# 在 house.py 的 get_heatmap_dict() 方法中
def get_heatmap_dict(self):
    heatmaps = {}
    for i in range(21):
        heatmaps[i] = []
    
    # 提取墙体关键点（通道0-12）
    for p in self.points:
        cord, _, p_type = p
        channel = self.get_number(p_type)  # 根据点类型确定通道
        heatmaps[channel-1].append((x, y))
    
    # 提取门窗角点（通道13-16）
    channel = 13
    for corner_type in ['left', 'right', 'up', 'down']:
        for corner in self.opening_corners[corner_type]:
            heatmaps[channel].append((x, y))
        channel += 1
    
    # 提取图标角点（通道17-20）
    for corner_type in ['upper_left', 'upper_right', 'lower_left', 'lower_right']:
        for corner in self.icon_corners[corner_type]:
            heatmaps[channel].append((x, y))
        channel += 1
```

### 2. 训练阶段（转换为热图张量）

在数据加载时，将关键点坐标转换为热图张量：

```python
# 在 augmentations.py 的 DictToTensor 类中
def cubi(self, sample):
    heatmaps = sample['heatmaps']  # 字典格式：{channel: [(x1,y1), (x2,y2), ...]}
    
    # 创建21通道的热图张量
    heatmap_tensor = np.zeros((21, height, width))
    
    # 在关键点位置设置为1
    for channel, coords in heatmaps.items():
        for x, y in coords:
            heatmap_tensor[int(channel), int(y), int(x)] = 1
    
    # 应用高斯模糊，使热图更平滑
    kernel = svg_utils.get_gaussian2D(int(30*scale))
    for i, h in enumerate(heatmap_tensor):
        heatmap_tensor[i] = cv2.filter2D(h, -1, kernel)
    
    # 结果：每个通道的值在 [0, 1] 之间，峰值在关键点位置
```

**高斯模糊的作用**：
- 将离散的关键点转换为连续的概率分布
- 使模型更容易学习（梯度更平滑）
- 允许关键点位置有小的误差

### 3. 模型输出

模型输出的前21个通道通过 **Sigmoid** 激活，输出值在 [0, 1] 之间：

```python
# 在 hg_furukawa_original.py 中
out[:, :21] = self.sigmoid(out[:, :21])  # 前21个通道应用Sigmoid
```

## 热图的使用

### 1. 提取墙体多边形

```python
# 在 post_prosessing.py 的 get_polygons() 函数中
wall_heatmaps = heatmaps[:13]  # 使用前13个通道
# 从热图中提取局部最大值作为墙体关键点
wall_points = get_wall_lines(wall_heatmaps, ...)
# 根据关键点构建墙体线段
wall_lines = get_wall_lines(...)
# 将线段转换为多边形
walls = extract_wall_polygon(...)
```

### 2. 提取门窗多边形

```python
# 使用通道13-16的门窗角点
openings = get_opening_polygon(heatmaps, walls, ...)
```

### 3. 提取图标多边形

```python
# 使用通道17-20的图标角点
icons = get_icon_polygon(heatmaps, icon_seg, ...)
```

## 热图 vs 分割图

| 特性 | 热图（Heatmap） | 分割图（Segmentation） |
|------|----------------|----------------------|
| **输出通道数** | 21个 | 12个（房间）+ 11个（图标） |
| **激活函数** | Sigmoid | Softmax |
| **值域** | [0, 1] | [0, 1]（概率分布） |
| **用途** | 检测关键点位置 | 像素级分类 |
| **输出含义** | 每个像素存在关键点的概率 | 每个像素属于某个类别的概率 |
| **后处理** | 提取局部最大值作为关键点 | Argmax得到类别ID |

## 总结

**热图是模型用于检测关键点的输出**，包括：
- ✅ 墙体关键点（13个通道）
- ✅ 门窗角点（4个通道）
- ✅ 图标角点（4个通道）

这些关键点用于：
1. 构建墙体线段和多边形
2. 定位门窗位置
3. 定位图标（家具、设备）位置
4. 生成矢量多边形分割结果

热图与分割图配合使用，共同完成户型图的完整解析：
- **热图**：提供结构信息（墙体、门窗、图标的位置）
- **分割图**：提供语义信息（房间类型、图标类型）
